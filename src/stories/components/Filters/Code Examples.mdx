# `Filters`

The Filters component is a dynamic component that
is responsible for displaying the filters available to the users.

**Dynamic Options Rendering:**

- Automatically updates and displays selected filters based on user requests

<Canvas of={FiltersStories.Primary} />

## Basic example

To view the examples and see this component in action

- **Search** - append your URL with `?q=shirt`
- **Browse** - append your URL with `/group_id/1035`

```jsx
import { CioPlp, CioPlpGrid, Filters } from '@constructor-io/constructorio-ui-plp';
import '@constructor-io/constructorio-ui-plp/styles.css';

const DEMO_API_KEY = 'key_M57QS8SMPdLdLx4x';

export default function MyComponent() {
  return (
    <CioPlp apiKey={DEMO_API_KEY}>
      <CioPlpGrid>
        {(props) => {
          return <Filters facets={props.filters.facets} />;
        }}
      </CioPlpGrid>
    </CioPlp>
  );
}
```

## Render Props pattern

To override the default rendering of the Filters component, you may pass a Render Props function to the component.

```jsx
import { CioPlp, CioPlpGrid, Filters } from '@constructor-io/constructorio-ui-plp';
import '@constructor-io/constructorio-ui-plp/styles.css';

const DEMO_API_KEY = 'key_M57QS8SMPdLdLx4x';

function MyCustomFilters(props) {
  const { facets, setFilter } = props;
  return <pre>{JSON.stringify(facets, null, 2)}</pre>;
}

export default function MyComponent() {
  return (
    <CioPlp apiKey={DEMO_API_KEY}>
      <CioPlpGrid>
        {(props) => {
          return (
            <Filters facets={props.filters.facets}>
              {(filterProps) => {
                return <MyCustomFilters {...filterProps} />;
              }}
            </Filters>
          );
        }}
      </CioPlpGrid>
    </CioPlp>
  );
}
```

## Hiding Facets and Options

You can hide facets or individual facet options using three methods (📦 v1.9.0 required):

> **Note:** This functionality is intended for **conditional rendering per page**. For example, showing a facet on search results but hiding it on category pages. If a facet or option should be hidden across your entire site, use the hidden facets feature in the Constructor.io dashboard or [API](https://docs.constructor.com/reference/configuration-facets) instead. Hidden facets configured through the dashboard are excluded from API responses entirely, which is more efficient than client-side filtering.

### Method 1: Metadata Field

Add `cio_plp_hidden: true` to the `data` field of facets or facet options in your Constructor.io dashboard.
These items will be automatically hidden without requiring custom functions.

```json
{
  "name": "price",
  "displayName": "Price",
  "type": "range",
  "data": {
    "cio_plp_hidden": true
  }
}
```

<Canvas of={FiltersStories.HiddenViaMetadata} />

### Method 2: Custom Field Getter

You can also customize the metadata field used for hiding by providing a custom `itemFieldGetters` to `CioPlpProvider`.
Each getter must return a boolean: `true` to hide the item, `false` to render it.

```jsx
<CioPlp
  apiKey={DEMO_API_KEY}
  itemFieldGetters={{
    getIsHiddenFilterField: (facet) => facet.data?.myCustomField,
    getIsHiddenFilterOptionField: (option) => option.data?.myCustomField,
  }}>
  {/* ... */}
</CioPlp>
```

### Method 3: Custom Function

Each function must return a boolean: `true` to hide the facet or option, `false` to render it.

#### Hiding Entire Facets

Use `isHiddenFilterFn` to hide entire facet groups. In this example, the "Price" facet is hidden:

```jsx
import { CioPlp, CioPlpGrid, Filters } from '@constructor-io/constructorio-ui-plp';
import '@constructor-io/constructorio-ui-plp/styles.css';

const DEMO_API_KEY = 'key_M57QS8SMPdLdLx4x';

export default function MyComponent() {
  // Hide the "price" facet entirely
  const isHiddenFilterFn = (facet) => facet.name === 'price';

  return (
    <CioPlp apiKey={DEMO_API_KEY}>
      <CioPlpGrid>
        {(props) => {
          return (
            <Filters
              facets={props.filters.facets}
              isHiddenFilterFn={isHiddenFilterFn}
            />
          );
        }}
      </CioPlpGrid>
    </CioPlp>
  );
}
```

<Canvas of={FiltersStories.HiddenFilters} />

#### Hiding Specific Options

Use `isHiddenFilterOptionFn` to hide specific options within facets. In this example, the "Black" and "Blue" color options are hidden:

```jsx
import { CioPlp, CioPlpGrid, Filters } from '@constructor-io/constructorio-ui-plp';
import '@constructor-io/constructorio-ui-plp/styles.css';

const DEMO_API_KEY = 'key_M57QS8SMPdLdLx4x';

export default function MyComponent() {
  // Hide specific color options
  const isHiddenFilterOptionFn = (option) =>
    option.value === 'Black' || option.value === 'Blue';

  return (
    <CioPlp apiKey={DEMO_API_KEY}>
      <CioPlpGrid>
        {(props) => {
          return (
            <Filters
              facets={props.filters.facets}
              isHiddenFilterOptionFn={isHiddenFilterOptionFn}
            />
          );
        }}
      </CioPlpGrid>
    </CioPlp>
  );
}
```

## Visual Filters

The Filters component can render color swatches and image thumbnails instead of plain text checkboxes using the shared `FilterOptionVisual` component. There are three ways to configure visual filters, listed below in order of priority.

### Method 1: Metadata Field

Set `cio_render_visual: true` on the facet's `data` in the Constructor dashboard, and `hex_color` or `image_url` on each option's `data`.

```json
{
  "name": "color",
  "displayName": "Color",
  "type": "multiple",
  "data": {
    "cio_render_visual": true
  },
  "options": [
    {
      "value": "Red",
      "data": {
        "hex_color": "#FF0000"
      }
    },
    {
      "value": "Floral",
      "data": {
        "image_url": "https://example.com/floral-pattern.jpg"
      }
    }
  ]
}
```

<Canvas of={FiltersStories.VisualColorFilters} />

### Method 2: Callback Functions

Use `isVisualFilterFn` to designate which facets render visually, and `getVisualColorHex` / `getVisualImageUrl` to resolve visual data per option.

```jsx
import { CioPlp, CioPlpGrid, Filters } from '@constructor-io/constructorio-ui-plp';
import '@constructor-io/constructorio-ui-plp/styles.css';

const colorHexMap = {
  Black: '#000000',
  Blue: '#0000FF',
  Red: '#FF0000',
  // ...
};

export default function MyComponent() {
  return (
    <CioPlp apiKey={DEMO_API_KEY}>
      <CioPlpGrid>
        {(props) => (
          <Filters
            facets={props.filters.facets}
            isVisualFilterFn={(facet) => ['color', 'Base Color'].includes(facet.name)}
            getVisualColorHex={(option) => colorHexMap[option.value]}
          />
        )}
      </CioPlpGrid>
    </CioPlp>
  );
}
```

<Canvas of={FiltersStories.VisualFilterViaCallback} />

### Method 3: Per-Facet Config (`filterConfigs`)

Use `filterConfigs` to enable visual rendering per facet. This has the highest priority and overrides both callbacks and metadata.

```jsx
import { CioPlp, CioPlpGrid, Filters } from '@constructor-io/constructorio-ui-plp';
import '@constructor-io/constructorio-ui-plp/styles.css';

export default function MyComponent() {
  return (
    <CioPlp apiKey={DEMO_API_KEY}>
      <CioPlpGrid>
        {(props) => (
          <Filters
            facets={props.filters.facets}
            filterConfigs={{ color: { renderVisual: true }, 'Base Color': { renderVisual: true } }}
          />
        )}
      </CioPlpGrid>
    </CioPlp>
  );
}
```

### Priority Order

The visual filter configuration is resolved in the following priority order:

**Facet-level (should this facet render visually?):**
1. `filterConfigs[name].renderVisual` — explicit per-facet override
2. `isVisualFilterFn(facet)` — callback
3. `facet.data.cio_render_visual` — metadata

**Option-level (what visual to show?):**
1. `getVisualImageUrl(option)` — callback (image takes priority)
2. `getVisualColorHex(option)` — callback
3. `option.data.image_url` — metadata
4. `option.data.hex_color` — metadata
5. Falls back to standard text filter if none match
